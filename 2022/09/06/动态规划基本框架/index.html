    斐波那契数列

    递推公式:

    f(n)=f(n-1)+f(n-2)

    f(0)=0,f(1)=1

    public int fib(int n) {
    if (n == 0) {
        return 0;
    }
    if (n == 1) {
        return 1;
    }
    int[] dp = new int[n+1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i < n+1; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    return dp[n];
}

    零钱兑换

    给你一个整数数组 coins ，表示不同面额的硬币；以及一个整数 amount ，表示总金额。

    计算并返回可以凑成总金额所需的最少的硬币个数 。如果没有任何一种硬币组合能组成总金额，返回 -1 。

    你可以认为每种硬币的数量是无限的。

    自顶向下(递归)

    备忘录

    • 使用 memo 数组保存已经递归出结果的值(剪枝)

    • memo 数组初始化

      • 最大值为 amount + 1(没有比面值为1的硬币更小的)

    无解判断

    • 若遍历完所有可能后, memo 值还是最大值, 则置为 -1 表示无解
    private int traverse(int[] coins, int amount, int[] memo) {
    if (memo[amount] != amount + 1) {
        return memo[amount];
    }
    for (int i = 0; i < coins.length; i++) {
        if (amount - coins[i] >= 0) {
            int num = traverse(coins, amount-coins[i], memo);
            if (num == -1) continue;
            if (memo[amount] > num + 1) {
                memo[amount] = num + 1;
            }
        }
    }
    if (memo[amount] == amount + 1) {
      memo[amount] = -1;
    }
    return memo[amount];
}

public int coinChange(int[] coins, int amount) {
    int[] memo = new int[amount+1];
    for (int i = 0; i < amount + 1; i++) {
        memo[i] = i+1;
    }
    memo[0] =  0;
    int result = traverse(coins, amount, memo);
    return result;
}

    自低向上(动态规划)

    f(coin)=1,coin in coins

    f(coin+f(n))=f(n)+1, coin in coins, f(n) != n+1

    版本一

    public int coinChange(int[] coins, int amount) {
    boolean[] visited = new boolean[amount+1];
    if (amount == 0) return 0;
    int[] dp = new int[amount+1];
    for (int i = 0; i < amount + 1; i++) {
        dp[i] = i+1;
    }
    Queue<Integer> q = new LinkedList<>();
    for (int i = 0; i < coins.length; i++) {
        if (coins[i] <= amount) {
            dp[coins[i]] = 1;
            q.offer(coins[i]);
        }
    }
    while (!q.isEmpty()) {
        long cur = q.poll();
        for (int i = 0; i < coins.length; i++) {
            long coinL = coins[i];
              // 部分case cur+coin 会超出 int 范围, 所以需要使用 long
            long target = cur+coinL;
            if (target > amount) continue;
            if (dp[(int)cur] + 1 < dp[(int)target]) {
                dp[(int)target] = dp[(int)cur] + 1;
                  // 剪枝
                if (!visited[(int)target]) {
                    q.offer((int)target);
                    visited[(int)target] = true;
                }
            }
        }
    }
    return dp[amount] == amount + 1 ? -1 : dp[amount];
}

    可以看出这个版本缺点很明显, 需要额外的存储空间, 而且需要考虑 int 范围问题

    版本二

    调整下遍历逻辑, 外层循环从 1 遍历到 amount, 内层循环所有可能的选择.

    public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount+1];
    dp[0] = 0;
    for (int i = 1; i < amount + 1; i++) {
        dp[i] = i + 1;
    }
    for (int i = 1; i < amount + 1; i++) {
        for (int coin: coins) {
            if (i - coin < 0) continue;
            if (dp[i-coin] == i-coin+1) continue;
            dp[i] = Math.min(dp[i-coin]+1, dp[i]);
        }
    }
    return dp[amount] == amount + 1 ? -1 : dp[amount]; 
}
